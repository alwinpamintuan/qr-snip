# Support

Use the repository's issue forms for help with QR Snip:

- **Bug report:** reproducible application behavior that appears incorrect
- **QR compatibility report:** a QR design or rendering condition that does not decode as expected
- **Feature request:** a user problem or capability proposal

Before opening an issue:

1. Search existing issues for the same browser, QR condition, or failure.
2. Test the latest published release or current `main` build.
3. Read the compatibility and known-limitations sections in [README.md](README.md).
4. Reproduce the problem with a synthetic or non-sensitive QR code whenever possible.

Include the operating system, browser and version, browser zoom, display scaling, installation method, and exact reproduction steps. Do not post QR codes containing credentials, private links, identity data, Wi-Fi secrets, tickets, or other sensitive information.

Security vulnerabilities must not be reported through a public issue. Use the repository's [private vulnerability reporting form](https://github.com/alwinpamintuan/qr-snip/security/advisories/new) and follow [.github/SECURITY.md](.github/SECURITY.md).
